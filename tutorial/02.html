Part 2 - Non-JS Fallback

Before we implement the slider functionality, let's make sure things will look okay for people without javascript enabled. The easiest thing to do (in terms of not breaking the rest of our page layout) is to show the first slide and hide all the rest.

To achieve this, we hide all slides, then show the "first" one. This requires adding a "first" class to the first slide.
Note: it may be tempting to use the *:first-child pseudo-selector so you can avoid adding a class to the first slide,
but don't do this -- it will cause problems with the "infiniteSlide" feature later on. (Also, it's incompatible with IE6/7.)
